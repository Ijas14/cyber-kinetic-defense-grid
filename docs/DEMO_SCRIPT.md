# CKDG Demo Video Script

**Target length:** 2–3 minutes  
**Format:** Screen recording + voiceover  
**Tone:** Calm, technical, confident — not sales-y

---

## Pre-Recording Checklist

- [ ] Backend running: `cargo run --release`
- [ ] Lemonade running with Qwen 3.5 2B Q4 loaded
- [ ] Browser open at `http://localhost:3000` — system in **nominal state** (green)
- [ ] Hardware Panel visible — VRAM, model name, latency metrics all showing
- [ ] Browser zoom at 100%, window maximised
- [ ] Terminal hidden — nothing behind the UI
- [ ] No notifications, DND mode on

## Recording Tips

### Pacing
- Speak **slower than feels natural**. Screen recordings always feel rushed on playback.
- Pause for 2 seconds after clicking an attack button. Let the UI react before narrating.
- Do not narrate and click simultaneously — action first, then narrate.

### What to emphasise visually
| Moment | Slow down / zoom here |
|---|---|
| REALITY VIOLATION banner fires | Let it sit for 2s before speaking |
| Reality vs Telemetry populates | Pan slowly across all three columns |
| First token arrives in Physics tab | Let streaming be visible for 3–4s |
| Hardware Panel metrics | Zoom in — judges need to see the numbers |

### What NOT to show
- Terminal / cargo output
- Settings modal (unless you want to demo model switching)
- The attack buttons themselves up-close — they're a demo mechanism, not a product feature
